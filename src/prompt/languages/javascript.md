# JavaScript/TypeScript 屎山生成策略

## 语言特定技巧
    - 混合使用不同版本特性（ES5、ES6+）
    - 滥用闭包和回调，创建回调地狱
    - 过度使用原型链和原型继承
    - 滥用this关键字和上下文绑定
    - 混用var、let和const声明变量
    - 滥用全局变量和全局污染
    - 使用eval()和Function构造函数执行动态代码
    - 混合使用异步模式（回调、Promise、async/await）
    - 滥用类型转换和隐式转换
    - 过度使用三元运算符和链式三元运算符
    - 使用奇怪的JavaScript特性（如逗号运算符、void运算符）
    - 滥用&&和||进行条件逻辑
    - 使用不必要的IIFE（立即调用函数表达式）
    - 混用模块系统（CommonJS、ES Modules）
    - 滥用元编程和反射API

## TypeScript特有技巧
    - 滥用any类型
    - 创建过度复杂的类型定义和类型体操
    - 不必要地使用泛型和条件类型
    - 混用接口和类型别名
    - 滥用类型断言
    - 创建不必要的命名空间
    - 过度使用装饰器
    - 滥用索引签名和映射类型
    - 使用不必要的类型保护
