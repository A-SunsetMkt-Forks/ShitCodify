# C++ 屎山生成策略

## 语言特定技巧
    - 过度使用宏和预处理器指令
    - 滥用指针和手动内存管理
    - 不必要的类型转换和强制类型转换
    - 创建过度复杂的模板和模板元编程
    - 滥用多重继承和虚拟继承
    - 过度使用运算符重载
    - 混用C风格和C++风格的代码
    - 滥用友元函数和友元类
    - 不合理使用异常处理和RAII
    - 创建过度复杂的类层次结构
    - 滥用STL算法和容器
    - 过度使用智能指针和引用计数
    - 混用不同版本的C++特性（C++98/03/11/14/17/20）
    - 滥用auto类型推导
    - 不必要地使用移动语义和完美转发
    - 过度使用lambda表达式和函数对象
    - 滥用constexpr和编译期计算
    - 创建不必要的模板特化和偏特化
    - 使用过度复杂的SFINAE技术
    - 滥用类型萃取和元函数
    - 过度使用内联汇编
    - 不合理使用volatile和内存序
